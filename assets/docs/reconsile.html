📋 Reconcile Plugin

Import and reconcile member data from spreadsheets. Upload a CSV or XLSX file, and Reconcile will create new members or update existing ones — matching Home Groups and Intergroup Positions automatically by name.

What Reconcile Does

Reconcile adds an Import page under the Intergroup menu. You upload a spreadsheet containing member data, and the plugin reads each row, maps the columns to member fields, resolves text names (like "Tuesday Night Group" or "Treasurer") to their matching WordPress posts, and either creates a new member or updates the existing one.

It includes a dry run mode so you can validate your spreadsheet before committing any changes.

Upload spreadsheet → Map columns → Resolve names → Validate rows → Create / Update

Requirements

  • WordPress 6.0 or later
  • PHP 8.0 or later
  • Unity plugin — must be installed and activated (provides the member, group, and position data layer)
  • Scrutiny plugin — listed as a required dependency for GDPR compliance
  • PHP zip extension — required for XLSX file support (most servers have this by default)

Installation

  1. Make sure Unity and Scrutiny are installed and activated.
  2. Upload the reconcile folder to /wp-content/plugins/, or install via the WordPress plugin uploader.
  3. Go to Plugins → Installed Plugins and activate Reconcile.
  4. A new Import submenu appears under Intergroup.

The Import Page

Navigate to Intergroup → Import. The page has three sections:

  • Column reference table — shows every accepted column header, its aliases, and any special rules (dates, booleans, name lookups)
  • File upload form — choose a .csv or .xlsx file, toggle dry run on or off, and click Import
  • Results panel — appears after the import runs, showing counts and any warnings or errors
💡
Only users with the manage_options capability (typically Administrators) can access the Import page and run imports.

How Import Works

When you click Import, Reconcile processes your spreadsheet in this order:

  1. Read the file — CSV files are streamed line-by-line; XLSX files are parsed using PHP's built-in ZIP and XML extensions (no external library needed).
  2. Map columns — the first row is treated as headers. Each header is normalised (lowercased, trimmed) and matched against the known aliases for each property. If required columns are missing, the import stops with an error.
  3. Resolve names — Home Group and Intergroup Position values are matched (case-insensitive) against existing WordPress posts. Unmatched names generate warnings but don't block the import.
  4. Validate each row — Anonymous Name must not be empty. If an Intergroup Position is set, a valid Rotation Date is required. Invalid rows are skipped with a reason.
  5. Create or update — Reconcile looks for an existing member by matching the Anonymous Name. If found, the member is updated. If not, a new member post is created. Existing fields that are not part of the import (like profile text or display settings) are preserved on updates.

Dry Run Mode

The "Dry run" checkbox is enabled by default. When checked, Reconcile validates every row and reports what would happen — how many members would be created, updated, or skipped — without actually writing anything to the database.

✅
Always run a dry run first. This catches missing columns, unrecognised group/position names, and invalid dates before any data is changed. Once you're happy with the dry run results, uncheck the box and import again for real.

File Formats

Reconcile accepts two file types:

FormatExtensionNotes
CSV.csvComma-separated values. UTF-8 encoding recommended. BOM is stripped automatically if present.
Excel.xlsxModern Excel format. Only the first worksheet (sheet1) is read. Requires the PHP zip extension.

In both cases, the first row must contain column headers. All subsequent rows are treated as data.

Column Matching

Reconcile normalises your column headers before matching: it lowercases the text, trims whitespace, and strips special characters (keeping only letters, numbers, spaces, and underscores). This means all of the following would match the "Anonymous Name" property:

  • Anonymous Name
  • anonymous_name
  • ANONYMOUS NAME
  • anonymous name

Extra columns in your spreadsheet that don't match any known property are silently ignored — they won't cause errors.

Home Group Lookup

The Home Group column should contain the group name as text (e.g. "Tuesday Night Group"). Reconcile loads all existing groups into a cache and matches your text against group titles, case-insensitively.

  • If a match is found, the group's post ID is assigned to the member.
  • If no match is found, the Home Group is set to 0 (unassigned) and a warning is added to the results listing all unresolved group names.
  • An empty Home Group cell is valid — it simply means no group is assigned.

Position Lookup

The Intergroup Position column should contain the position's long name as text (e.g. "Treasurer", "GSR Liaison"). Reconcile matches against position long names, case-insensitively.

  • If a match is found, the position's post ID is assigned.
  • If no match is found, the position is set to 0 and a warning is generated.
  • An empty position cell is valid — it means the member holds no intergroup position.
🚨
If a Position is set, a valid Position Rotation date is required. Rows with a position but no rotation date (or an invalid one) are skipped entirely.

Date Formats

The Position Rotation column accepts these date formats. All separators (/ - .) are interchangeable:

FormatExample
yyyy/MM/dd2025/01/15   2025-01-15   2025.01.15
dd/MM/yyyy15/01/2025   15-01-2025   15.01.2025
dd/MM/yy15/01/25   15-01-25   15.01.25

All dates are normalised to yyyy-MM-dd (ISO format) on import. Two-digit years are interpreted as 2000+, so 25 becomes 2025. The date must be a real calendar date — 31/02/2025 will be rejected.

GSR Boolean Values

The GSR Status column is parsed as a boolean. The following values (case-insensitive) are treated as true:

yes y true 1 gsr

Everything else — including empty cells, "no", "n", "false", "0" — is treated as false.

Understanding Results

After an import (or dry run), the results panel shows a summary with four counts:

MetricMeaning
Total RowsThe number of data rows in the spreadsheet (excluding the header row).
CreatedNew member posts that were created (or would be created in dry run).
UpdatedExisting members that were matched by Anonymous Name and updated.
SkippedRows that could not be imported due to validation errors.

Skipped Rows

When a row is skipped, Reconcile reports the row number, the reason for skipping, and a detail view showing the raw cell values and any resolved IDs. Common skip reasons:

Anonymous Name is empty

Every member must have an Anonymous Name. Rows with a blank name are skipped.

Position set but Rotation missing or invalid

If an Intergroup Position value is provided, a valid Rotation Date is required in the same row. If the date cell is empty, blank, or in an unrecognised format, the row is skipped.

Failed to create/update WordPress post

Rare, but can happen if there's a database error or permissions issue. The error message from WordPress is included in the skip reason.

Warnings

Warnings don't prevent an import — they flag things you should review:

  • Unresolved Home Groups — group names that didn't match any existing group. The member is imported with Home Group set to 0 (unassigned). Check the spelling matches the group title in WordPress exactly.
  • Unresolved Intergroup Positions — position names that didn't match. Same as groups — check the spelling against the position's long name in WordPress.
  • Per-row lookup warnings — a specific row's Home Group or Position couldn't be matched. Shown inline with the row number.

Troubleshooting

"Missing required columns" error

Your spreadsheet header row doesn't contain all required columns. Check the Column Reference for accepted header names. Remember: the first row must be headers, not data.

All rows are being skipped

Check that Anonymous Name is populated in every row. If you're providing Intergroup Positions, make sure every row with a position also has a valid rotation date.

Group/Position names aren't matching

Matching is case-insensitive but otherwise exact. Check for extra spaces, typos, or abbreviated names. The group name must match the post title in WordPress; the position name must match the position's long name.

XLSX files fail with "zip extension required"

The PHP zip extension is needed to read XLSX files. Ask your hosting provider to enable it, or convert your file to CSV as a workaround.

CSV encoding issues (garbled characters)

Save your CSV in UTF-8 encoding. Reconcile strips the UTF-8 BOM automatically, but other encodings (like Windows-1252) may cause garbled text.

Import page not visible

The Import page requires manage_options — only Administrators can see it. It also requires both Unity and Scrutiny to be active, and appears under the Intergroup menu (added by Amber).

FAQ

Does importing overwrite existing member data?

Yes — for the fields included in the import. If an existing member is found (by Anonymous Name), the imported values replace the current ones for those fields. However, fields that are not part of the import (such as profile text, display settings, and meeting PO) are preserved.

How does Reconcile match existing members?

By Anonymous Name (exact match). If a member with the same Anonymous Name already exists in WordPress, that member is updated. If no match is found, a new member is created.

Can I import members without a Home Group or Position?

Yes. The columns must exist in the spreadsheet (they're structurally required), but the cells can be left empty. An empty Home Group or Position simply means "none assigned".

What happens to the uploaded file?

The file is moved to a temporary directory inside wp-content/uploads/reconcile-tmp/, processed, and then deleted immediately — whether the import succeeds or fails. No spreadsheet data is stored permanently.

Can I undo an import?

There is no built-in undo. If you've made a mistake, you'll need to correct the data manually or re-import a corrected spreadsheet. This is why the dry run mode exists — always validate first.

Can I use Reconcile without Amber?

Reconcile registers its menu page under the "Intergroup" menu slug, which is created by Amber. If Amber is not active, the Import page won't appear in the menu. You need at minimum Unity, Scrutiny, and Amber for the full menu structure.
